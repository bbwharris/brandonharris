I always recommend Active Scaffold to fellow Rails developers. There is a learning curve, but it can cut a lot of development time out of your clients custom CMS.

Paperclip has pretty much dominated the Rails attachment market for the past year. My company has migrated to it, and there are countless tutorials and testimonials regarding it’s ease of use.

I am assuming that you are using the latest version of Rails (2.3.5 as of this writing), Active Scaffold, and Paperclip. First we will install all the necessary components, then we will configure the rails app.

Install Active Scaffold:

{% highlight console %} script/plugin install git://github.com/activescaffold/active_scaffold.git {% endhighlight %}

Active Scaffold is big so this takes a while.

Install Paperclip:

{% highlight console %} script/plugin install git://github.com/thoughtbot/paperclip.git {% endhighlight %}

As of 6/14/2010 this is no longer needed, the bridge has been moved to the core

Lets create a basic person model:

{% highlight console %} script/generate model Person {% endhighlight %}

To create controllers in the admin namespace, I don’t use the rails controller generator. I don’t like cluttering up my projects with files I will never use, since I employ Active Scaffold for 99% of my admin interfaces. This is my personal preference, you may do this anyway that you wish.

{% highlight console %} mkdir app/controllers/admin && touch app/controllers/admin/people_controller.rb {% endhighlight %}

Lets configure the “Person” migration for paperclip.

{% highlight ruby linenos %} class CreatePeople < ActiveRecord::Migration def self.up create_table :people do |t| t.string :name, :avatar_file_name, :avatar_content_type t.integer :avatar_file_size t.datetime :avatar_updated_at t.timestamps end end def self.down drop_table :people end end {% endhighlight %}

Configure the Person model to accept the paperclip attachment.

{% highlight ruby linenos %} class Person < ActiveRecord::Base has_attached_file :avatar, :styles => {:thumbnail => "75x75#"} end {% endhighlight %}

Setup your routing for the admin namespaced “People” controller.

{% highlight ruby linenos %} ActionController::Routing::Routes.draw do |map| map.namespace :admin do |admin| admin.resources :people, :active_scaffold => true end map.connect ':controller/:action/:id' map.connect ':controller/:action/:id.:format' end {% endhighlight %}

Create a layout file for admin, I use Haml, and I recommend that you do too.

{% highlight haml linenos %} !!!STRICT %html{'xmlns' => "http://www.w3.org/1999/xhtml", :lang => "en"} %head = javascript_include_tag :defaults = active_scaffold_includes %body #content = yield {% endhighlight %}

Finally, configure your active scaffolded controller. The key here is to include the attachment model name, in this case “avatar”.

{% highlight ruby linenos %} class Admin::PeopleController < ApplicationController layout 'admin' active_scaffold :person do |config| #restrict all columns to these three config.columns = [:name, :avatar] #include multipart for create and update forms config.create.multipart = true config.update.multipart = true end end {% endhighlight %}
